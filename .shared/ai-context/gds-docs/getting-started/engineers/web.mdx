# JavaScript

The JavaScript library includes React components as well as tokens that can be used directly in code or as CSS variables.

## Installation

<ArtifactoryHint />

Using your favorite npm package manager:

```bash
yarn/npm/pnpm install @grammarly/design-system
```

You'll also want to include the GDS styles within one of your CSS files:

```css
/** for modern bundlers **/
@import '@grammarly/design-system' 

/** OR for webpack (MiniCssExtractPlugin) **/ 
@import '~@grammarly/design-system/dist/index.css'
```

### Optional: fonts

You will also need the following if your app doesn't already provide the Inter and Matter fonts:

```css
import '@grammarly/design-system/dist/fonts.css'
```

If your app doesn't already set the font-family for the body, you may also need to add the following CSS:

```css
body {
  font-family: var(--font-stack-inter);
}
```

### Optional: shadow DOM styles

By default, GDS attaches CSS variables to `:root`. If your app uses shadow DOM, you probably also need them on `:host`:

```css
import '@grammarly/design-system/dist/shadow.css'
```

## Components

Components are provided via the `@grammarly/design-system` package. They are built with React and Typescript.

See [Components Status](/components/component-status) for the complete list of available components and relevant version information.

Sample usage:

```tsx
import { Tag } from "@grammarly/design-system";

<Tag variant="warning" label="Beta" />;
```

## Tokens

The design system tokens are available as individual values in `@grammarly/design-system`. TypeScript types are also included.

<CodeBlock language="ts" children={`import { Tokens } from '@grammarly/design-system'`} />

CSS variables are also available, as demonstrated in the following sections.

### Border Radius

#### CSS

```css
.someSelector {
  border-radius: var(--radius-1);
}
```

#### JavaScript

```tsx
import { Tokens } from "@grammarly/design-system";

function Example() {
  return (
    <div
      style={{
        borderRadius: `${Tokens.Radius.RadiusHalf}rem`,
      }}
    ></div>
  );
}
```

### Color

Most designs will use the <Link to="/foundations/color">Semantic palette</Link> with some additional colors pulled from the Core palette when necessary.

#### CSS

```css
.someSelector {
  background: var(--color-background-base-default); /* semantic palette */
  color: var(--blue-40); /* core palette */
}
```

#### JavaScript

```tsx
import { Tokens } from "@grammarly/design-system";

function Example() {
  return (
    <div
      style={{
        background: Tokens.SemanticColor.Color.Background.Base.Default,
        color: Tokens.Color.Blue40,
      }}
    ></div>
  );
}
```

### Elevation

#### CSS

```css
.elevation-high {
  border: var(--elevation-high-border);
  box-shadow: var(--elevation-high-shadow);
}
```

#### JavaScript

```tsx
import { Tokens } from "@grammarly/design-system";

const styles = {
  boxShadow: Tokens.Elevation.ElevationHigh.boxShadow,
  border: Tokens.Elevation.ElevationHigh.border,
};
```

### Iconography

React-components based on `.svg` assets are available. All icons are `aria-hidden=true` and `role="img"` by default. See [all available icons](/foundations/iconography).

#### Examples

Basic usage:

<CodeBlock
  language="tsx"
  children={`import { EmojiAnalyticalIcon } from 'grammarly/design-system'\n\n<EmojiAnalyticalIcon />`}
/>

Override any SVG element props:

<CodeBlock
  language="tsx"
  children={`import { EmojiAnalyticalIcon } from 'grammarly/design-system'\n\n<EmojiAnalyticalIcon color fill viewBox width height ...any svg props... />`}
/>

Override colors on some state (hover, active, etc.):

<CodeBlock
  language="tsx"
  children={`import { InterfaceCopyIcon } from 'grammarly/design-system'\n\n// variant 1 — JS API:\n<InterfaceCopyIcon fill="someColorHere" />\n\n// variant 2 — pass classname and contol it via CSS:\n<InterfaceCopyIcon className="someCssClassHere" />`}
/>

Add special a11y tags:

<CodeBlock
  language="tsx"
  children={`import { EmojiAnalyticalIcon } from 'grammarly/design-system'\n\n<EmojiAnalyticalIcon title titleId desc descId />`}
/>

### Typography

[GDS Typography](/foundations/typography) is accessible as React components ([Heading](/components/heading) and [Text](/components/text)).

The necessary fonts and `@font-face` rules are available with the following import (since version 6.11.0):

```css
import '@grammarly/design-system/dist/fonts.css'
```

The following CSS variables are also available as a convenience:

- `--font-stack-inter`
- `--font-stack-matter`

For example, you could set the default font for your app like this:

```css
body {
  font-family: var(--font-stack-inter);
}
```
