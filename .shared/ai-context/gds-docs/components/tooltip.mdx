## Usage

Use when a short, text-only, inline description of an element’s purpose is needed. [Learn how to write effective Tooltips](/components/tooltip#writing).


Because they are often used to describe an IconButton's purpose, [Tooltips are automatically built into the IconButton component](/components/icon-button#primary).

**Do not use when:**

- A user needs to take action or advance through steps. Instead, use a [Popover](/components/popover).
- An image or long, descriptive text is required. Instead, use a [Popover](/components/popover).
- The goal is to focus a user's attention on the content in a layer. Instead, use a [Modal](/components/modal).
- The content repeats visible UI text, such as a label on a [Button](/components/button).

### macOS, Windows, iOS, and Android

macOS, Windows, iOS, and Android use a native Tooltip, which behaves according to the rules set by the operating system. The design on these platforms does not need to match the Web component, but you can use design system tokens to style it. [View the design tokens](/tokens).

## Behavior

### Position

- By default, a Tooltip appears above the element that it’s paired with.
- If there isn’t enough space above an element to accommodate a Tooltip, it will automatically display in another available position. Position priority follows this order: top, right, bottom, left.
- The pointer is always centered on an element and touches its edge.

### Appearing and disappearing

- A Tooltip appears when a user hovers on or focuses on a paired element after a 0.2-second pause.
- A Tooltip disappears when the cursor is moved off of the element or when the tab focus is changed to a different element on the page.

### Size

The width and height of a Tooltip change based on content within it.

#### Width

The maximum width of a Tooltip is 200px. At its maximum width, the label and annotation can both wrap to the following line.

#### Height

- The height of a Tooltip is determined by the length of the content.
- The content container in a Tooltip does not scroll.

## Accessibility

A Tooltip should be accessible with the mouse cursor. For screen readers, content is read in this order:

1. The element
2. The Tooltip content
3. A keyboard shortcut, if applicable

### Keyboard interaction​

| Key                                  | Expected result                                                                                                                                      |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| <kbd>Space</kbd> or <kbd>Enter</kbd> | Activates an element’s Tooltip.                                                                                                                      |
| <kbd>Esc</kbd>                       | Dismisses the Tooltip but does not dismiss any related overlay UI, such as a Modal. When the Tooltip is closed, focus returns to the paired element. |

## Writing

### Tooltips

In as few words as possible, the content should make clear what the functionality or purpose of a paired element is. Provide only information that is not essential. It needs to be possible for a user to infer how to use a product without reading any Tooltips.

For more robust access to essential content, instead use plain UI text, a [Popover](/components/popover), or a [Modal](/components/modal).

### Keyboard shortcuts

An optional annotation can display a keyboard shortcut associated with an element.

- Make sure keyboard shortcuts are dynamic based on the operating system. For example, Option for Mac is Alt for Windows.
- Shorten: Ctrl, Alt, Del, Esc, Fn
- Spell out: Tab, Enter, Return, Shift, Option, End, Home, Windows key
- Use a symbol only for Command (⌘) and the up, right, down, or left arrow keys.
- Do not use spaces around the + in combination shortcuts.

## Installation

1. Import the CSS (if not done already).

```css
@import "@grammarly/design-system";
```

2. Import the Tooltip component in JS.

```tsx
import { Tooltip, TooltipContent, TooltipTrigger } from "@grammarly/design-system";
```
