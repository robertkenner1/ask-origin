# Grammarly Design System (GDS)

The Grammarly Design System empowers teams to build high-quality, cohesive product experiences at scale. Built and maintained by the Design Foundations team in collaboration with design and engineering communities.

**Package**: `@grammarly/design-system`
**Installation**: `npm install @grammarly/design-system` or `pnpm add @grammarly/design-system`
**Documentation**: https://uifoundation.gpages.io/grammarly-design-system
**Source**: https://gitlab.grammarly.io/uifoundation/grammarly-design-system

## Quick Start

```typescript
import { Button, TextField, Modal, Flex, Text } from '@grammarly/design-system';
import '@grammarly/design-system/dist/index.css';
```

## GDS Principles

- **Tools over rules**: Flexible toolbox that encourages creativity, not rigid rules
- **Everyone has a seat at the table**: Partners across disciplines collaborate on the system
- **Be completely inclusive**: Accessibility integrated into all aspects of design and code
- **Design with scale in mind**: Reusable components reduce technical and design debt

## Core Values

- **Inclusive**: Consider diverse perspectives for best outcomes
- **Intuitive**: Easy to understand for everyone
- **Impactful**: Create essential tools people can't work without
- **Transparent**: Work openly, welcome feedback, share often
- **Quality**: Deliver best-in-class solutions

## Common Patterns

### Basic Layout
```typescript
<Flex direction="column" gap={4} padding={6}>
  <Heading size="lg">Page Title</Heading>
  <Text>Content goes here</Text>
</Flex>
```

### Form Example
```typescript
<Form onSubmit={handleSubmit}>
  <Flex direction="column" gap={4}>
    <TextField label="Email" type="email" required />
    <TextField label="Password" type="password" required />
    <Button type="submit" text="Sign In" />
  </Flex>
</Form>
```

### Modal Pattern
```typescript
<Modal
  isOpen={isOpen}
  onClose={() => setIsOpen(false)}
  heading="Confirm Action"
>
  <Text>Are you sure you want to proceed?</Text>
  <Flex gap={2} justify="end">
    <Button variant="tertiary" text="Cancel" onClick={() => setIsOpen(false)} />
    <Button variant="primary" text="Confirm" onClick={handleConfirm} />
  </Flex>
</Modal>
```

## Best Practices

### DO ✅
- Use Flex component for layouts instead of custom CSS
- Use semantic HTML with GDS components
- Follow accessibility guidelines built into components
- Use design tokens for colors and spacing
- Prefer composition of small components

### DON'T ❌
- Don't create custom buttons - use Button component
- Don't use inline styles - use component props
- Don't hardcode colors - use design tokens
- Don't skip labels on form inputs
- Don't use divs for clickable elements

## Component Selection Guide

**Need a button?**
- Action → `Button`
- Navigation → `ButtonAsLink` or `Link`
- Icon only → `IconButton`

**Need text input?**
- Short text → `TextField`
- Long text → `Textarea`
- Search → `SearchField`
- Dropdown → `Select`
- Searchable dropdown → `Combobox`
- Code/PIN → `VerificationCode`

**Need a container?**
- Layout → `Flex`
- Generic wrapper → `Box`

**Need feedback?**
- Temporary → `Toast`
- Persistent → `Notification`
- Help text → `Tooltip`
- Confirmation → `Modal`
