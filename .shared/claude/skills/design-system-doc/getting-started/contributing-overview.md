# Contributing to Origin

We welcome and greatly appreciate contributions to Origin! Listed below are the resources and guidelines for contributing.

## Adding a component in code

If you wish to add a component to Origin, follow the steps below:

1. Discuss the potential contribution with the Origin team in [#ask-origin-design-system](https://app.slack.com/client/E06GHNPFX4L/C03MNHYDT5E)
2. Read the [contribution process guidelines](https://docs.google.com/document/d/1B4_Cqo_siEDr7NtJknb_YTSvnWD-flLisUA72WrHxnY/edit#heading=h.wlcpn9oii4kl)
3. Write a schema doc based on [this template](https://docs.google.com/document/d/1rq7s-HBxPbie-cUAdS25AVaFLNN75u87jOIN0ktKYSw/edit#heading=h.d72vdubtjukt)
4. Once the schema is written, share for review in #ask-origin-design-system
5. After the schema is approved, [development begins](https://docs.google.com/document/d/1B4_Cqo_siEDr7NtJknb_YTSvnWD-flLisUA72WrHxnY/edit#bookmark=id.ygk820dkei5x)!

Always feel free to ask questions along the way!

## Updating a current component

If you'd like to add a new variation of a component that's already implemented in code, or you find a bug you'd like to fix, please reach out in [#ask-origin-design-system](https://app.slack.com/client/E06GHNPFX4L/C03MNHYDT5E) and the Origin team will help coordinate your update.

## Proposing a new component design
